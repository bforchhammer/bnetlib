bnetlib

bnetlib is an object-oriented interface for the Battle.net REST-ish API. It tries to keep a direct mapping to the actual resource names and response values.

The connection itself will be done by third-party libraries. bnetlib ships with the following adapters: Aura, Buzz, ZendFramework and even a Stub adapter.

Entities are the heart of the library and presumably the main difference between other Battle.net libraries. They provide an object-oriented interface for the returned data, convenient methods, data conversion, locale support for non-localized data and can even be used to provide request arguments like Realm Names, Character Names or Item Ids. But if you prefer a lightweight array, simply set the return value to ::RETURN_PLAIN and the library will return the response as array (for JSON) or binary string (for Images).

Extending bnetlib is simple... trivially simple! The library uses various techniques to ensure that you can replace every single class with your own:

  • No hardcoded dependencies. Most dependencies are visible from the outside and if that is not possible to achieve, the library will make use of the Service Locator.
  • No type hints for actual classes. Just implement an interface and you are ready to go.
